# Git Stash 实战演示教程

## 🎯 什么是 Git Stash？

Git Stash 是一个临时存储功能，可以保存当前工作区和暂存区的修改，让你能够快速切换到其他任务，而不需要创建一个不完整的提交。

## 🔧 核心概念

### Stash 的本质
- **临时存储**: 将未提交的修改临时保存起来
- **工作区清理**: 让工作区回到干净状态
- **随时恢复**: 可以在任何时候恢复保存的修改

### 适用场景
1. **紧急切换分支**: 正在开发功能时突然需要修复bug
2. **临时实验**: 想要尝试不同的实现方案
3. **代码审查**: 需要切换到干净状态查看其他代码
4. **同步更新**: 需要pull最新代码但有未完成的修改

## 📋 演示场景规划

### 场景1: 紧急Bug修复
- 正在开发新功能
- 突然需要修复紧急bug
- 使用stash保存当前工作
- 切换分支修复bug
- 修复完成后恢复工作

### 场景2: 实验性修改
- 想要尝试不同的实现方案
- 保存当前方案
- 尝试新方案
- 对比选择最佳方案

### 场景3: 代码同步
- 本地有未完成的修改
- 需要pull远程最新代码
- 使用stash避免冲突
- 同步后恢复修改

### 场景4: 多个stash管理
- 管理多个临时保存
- 给stash添加描述
- 查看和选择性恢复
